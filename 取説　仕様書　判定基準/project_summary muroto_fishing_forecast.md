# 室戸岬沖 出船可否判断システム — 開発まとめ

**作成日**: 2026-04-10  
**対象プロジェクト**: 室戸岬沖 遊漁船 出船可否判断システム  
**現行バージョン**: v3（GitHub Pages + GitHub Actions 自動運用版）

---

## 1. プロジェクト概要・目的

室戸沖（高知県）への遊漁船の出船可否を、リアルタイムの海況・気象データから自動判定するシステム。

**最終目標**: 遊漁船予約管理システムの顧客に対して、出船可否通知を自動送信するための「元データ（判定結果）」を生成・提供すること。

- **対象海域**: 室戸沖（北緯 33.2° / 東経 134.2°）
- **データソース**: Open-Meteo API（無料・認証不要）
- **判定方式**: 6ファクタースコアリングによる4段階評価
- **予報期間**: 5日先まで（0時・6時・12時・18時 の4スロット/日）

## 4. データソース・取得仕様

### Open-Meteo Marine API

| フィールド | 単位 | 説明 |
|---|---|---|
| `wave_height` | m | 合成波高（風波＋うねり） |
| `wave_direction` | 度（0=北・時計回り） | 波の進行方向 |
| `wind_wave_height` | m | 風波高（短周期波） |

エンドポイント:
```
https://marine-api.open-meteo.com/v1/marine
  ?latitude=33.2&longitude=134.2
  &hourly=wave_height,wave_direction,wind_wave_height
  &timezone=Asia%2FTokyo&forecast_days=5
```

### Open-Meteo Weather API

| フィールド | 単位 | 説明 |
|---|---|---|
| `wind_speed_10m` | m/s | 地上10m高度の風速 |
| `wind_direction_10m` | 度（0=北・時計回り） | 風向 |
| `precipitation` | mm | 1時間降水量 |

エンドポイント:
```
https://api.open-meteo.com/v1/forecast
  ?latitude=33.2&longitude=134.2
  &hourly=wind_speed_10m,wind_direction_10m,precipitation
  &wind_speed_unit=ms&timezone=Asia%2FTokyo&forecast_days=5
```

### 取得・抽出スロット

毎日 **0時・6時・12時・18時**（JST）の4スロット × 最大5日分 = 最大20レコード

> 日次総合判定の代表値には **6:00 と 12:00 の rawScore 平均**を使用（出港〜沖での主要活動時間を代表）

## 5. 判定ロジック（v3 — 6ファクター）

### 基本コンセプト

波高の実測値（m）をベース値として使用し、各種条件補正を加減算。rawScore を閾値で分類して最終判定を出力する。

```
rawScore = wave_height + ②補正 + ③補正 + ④補正 + ⑤補正 + ⑥補正
最終判定 = rawScore を閾値で4段階分類
```

### 最終判定閾値（rawScore → 記号）

| rawScore | 記号 | スコア値 | 予約システム送信ステータス |
|---|---|---|---|
| 0.8 以下 | ◎ 釣り日和 | 3 | ✅ 出船可 |
| 0.8 より大〜1.2 以下 | ○ 出船可能 | 2 | ✅ 出船可 |
| 1.2 より大〜1.6 以下 | △ 未定・要確認 | 1 | ⚠ 要確認 |
| 1.6 より大 | × 出船不可 | 0 | ❌ 中止 |

---

### ファクター① 波高ベース値

波高（m）の実測値をそのままベース値として使用。変換・正規化は行わない。

---

### ファクター② 風向補正（室戸岬地形特性）

室戸岬沖は南〜南東方向からの風で沖合の波が成長しやすい地形特性がある。北風（陸風方向）は比較的安全。

| 条件 | 補正値 | タグ |
|---|---|---|
| 南系（135〜225°）かつ風速 > 6 m/s | **+0.5** | 南風↑（赤） |
| 北系（315〜45°） | ±0 | 北風◎（緑） |
| 上記以外（東西系） | **+0.25** | 東西風（橙） |

---

### ファクター③ 逆波補正（風向 vs 波向きの干渉）

風向と波向きが逆・直交する場合、三角波・急峻波が発生し体感リスクが増大する。

| 角度差 | 補正値 | タグ |
|---|---|---|
| 120° 以上（逆方向） | **+0.2** | 逆波（赤） |
| 90〜120°（交差方向） | **+0.1** | 交差波（橙） |
| 90° 未満（同方向系） | ±0 | — |

角度差の計算式:
```
diff = min(|windDir - waveDir| mod 360, 360 - |windDir - waveDir| mod 360)
```

波向きデータ（`wave_direction`）が取得できない場合、このファクターは省略。

---

### ファクター④ グレーゾーン複合補正

グレーゾーン波高帯（0.8m超）で②南風補正と③逆波補正が**同時に発生**した場合の追加悪化補正。

| 適用条件 | 補正値 | タグ |
|---|---|---|
| wave > 0.8m **かつ** ②南風補正発動 **かつ** ③逆波または交差波補正発動 | **+0.2** | 複合悪化（紫） |

---

### ファクター⑤ 風速トレンド補正

前スロット（6時間前）との風速変化を評価。増加傾向はリスク増、減少傾向はリスク低減。

| 条件 | 補正値 | タグ |
|---|---|---|
| 前スロット比 +0.5 m/s 以上の増加 | **+0.2** | 風↑（橙） |
| 前スロット比 −0.5 m/s 以上の減少 | **−0.2** | 風↓（緑） |
| 変化量 ±0.5 m/s 未満 | ±0 | — |

0時スロットの「前スロット」は前日18時のデータを参照。

---

### ファクター⑥ 風波補正（v3 新規追加）

風波（`wind_wave_height`）は短周期の急峻な波。小さければ安全側、大きければ危険側に補正。

| 条件 | 補正値 | タグ |
|---|---|---|
| 風波 ≤ 0.3 m（風波小） | **−0.3** | 風波小（緑） |
| 風波 ≥ 0.7 m（風波大） | **+0.3** | 風波大（赤） |
| 0.3m 超〜0.7m 未満 | ±0 | — |

風波データ（`wind_wave_height`）が取得できない場合、このファクターは省略。

---

### 補正タグ一覧

| タグ | 色 | 意味・補正値 |
|---|---|---|
| 南風↑ | 赤 | 南系風 > 6 m/s — 波成長リスク (+0.5) |
| 北風◎ | 緑 | 北系風 — 安全方位 (±0) |
| 東西風 | 橙 | 東西系風 — やや不利 (+0.25) |
| 逆波 | 赤 | 風と波が120°以上逆 — 三角波リスク (+0.2) |
| 交差波 | 橙 | 風と波が90〜120°交差 (+0.1) |
| 複合悪化 | 紫 | 南風＋逆波の同時発生 (+0.2) |
| 風↑ | 橙 | 前スロット比 +0.5 m/s 以上の増加傾向 (+0.2) |
| 風↓ | 緑 | 前スロット比 −0.5 m/s 以上の減少傾向 (−0.2) |
| 風波小 | 緑 | 風波 ≤ 0.3 m — 波質良好 (−0.3) |
| 風波大 | 赤 | 風波 ≥ 0.7 m — 急峻波リスク (+0.3) |

## 6. GitHub Actions 自動更新

`update-forecast.yml` が JST 6・9・12・15・18時に自動実行。

```yaml
on:
  schedule:
    - cron: '0 21,0,3,6,9 * * *'  # UTC（= JST 6/9/12/15/18時）
  workflow_dispatch:               # GitHub画面から手動実行も可能
```

### 実行フロー

```
Open-Meteo API（Marine + Weather）
    ↓ GitHub Actions が定期実行
fetch_forecast.py（Python スクリプト）
    ↓
forecast_data.json 生成
    ↓ git commit & push（自動コミット）
GitHub Pages が静的配信
    ↓
index.html が JSON を読み込み → 判定テーブル表示
```

フォールバック: `forecast_data.json` が未生成の場合、ブラウザが直接 Open-Meteo API を呼び出す。

### forecast_data.json のデータ構造

```json
{
  "updated": "2026-04-10T06:00:00Z",
  "rows": [
    {
      "t":        "2026-04-10T06:00",
      "wave":     0.92,
      "wind":     4.2,
      "dir":      180,
      "waveDir":  210,
      "windWave": 0.45,
      "rain":     0.1
    }
  ]
}
```

## 9. 注意事項・既知の制約

1. `wave_height` は合成波高（風波＋うねりの総和）。うねりが卓越する海況では実際より波高を過小評価する場合がある。
2. 予測精度は時間とともに低下。3日目以降の判定は参考値として扱う。
3. GitHub Actions の `GITHUB_TOKEN` はリポジトリの Settings → Actions → General → "Read and write permissions" が必要。
4. 突発的な気象変化（急発達低気圧など）はモデル予測に遅れが生じる場合がある。
5. `forecast_data.json` は GitHub Actions が自動生成・コミットするため、手動で編集・削除しないこと。
6. **船長判断が最優先**。本システムの出力はあくまで参考情報であり、最終判断は船長が行う。
