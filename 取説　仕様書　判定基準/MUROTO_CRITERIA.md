# 室戸沖 出船判断システム — 判定基準・データポイント仕様書

**バージョン**: v3.1  
**最終更新**: 2026-04-09  
**対象海域**: 室戸沖（北緯 33.2° / 東経 134.2°）

---

## 1. 取得データポイント

### 1-1. Open-Meteo Marine API

| フィールド名 | 単位 | 説明 |
|---|---|---|
| `wave_height` | m | 合成波高（風波＋うねり） |
| `wave_direction` | 度（0=北、時計回り） | 波の進行方向 |
| `wind_wave_height` | m | 風波高（風による短周期波） |

**API エンドポイント:**
```
https://marine-api.open-meteo.com/v1/marine
  ?latitude=33.2&longitude=134.2
  &hourly=wave_height,wave_direction,wind_wave_height
  &timezone=Asia%2FTokyo&forecast_days=5
```

### 1-2. Open-Meteo Weather API

| フィールド名 | 単位 | 説明 |
|---|---|---|
| `wind_speed_10m` | m/s | 地上10m高度の風速 |
| `wind_direction_10m` | 度（0=北、時計回り） | 風向 |
| `precipitation` | mm | 1時間降水量 |

**API エンドポイント:**
```
https://api.open-meteo.com/v1/forecast
  ?latitude=33.2&longitude=134.2
  &hourly=wind_speed_10m,wind_direction_10m,precipitation
  &wind_speed_unit=ms&timezone=Asia%2FTokyo&forecast_days=5
```

### 1-3. 取得時刻スロット

毎日 **0時・6時・12時・18時**（JST）の4スロット × 最大5日分 = 最大20行

> 日次総合判定には **6:00 と 12:00** の rawScore 平均を使用（出港〜沖での状況を代表する時間帯）

## 2. スコア計算方式（6ファクター）

### 基本コンセプト

**波高メートル値をそのままベース値として使用**し、各種条件補正を加算・減算する。  
「低い値 = 良い海況」という方向性で計算し、最終的に閾値分類で ◎○△× を判定する。

```
rawScore = wave_height + ②補正 + ③補正 + ④補正 + ⑤補正 + ⑥補正
最終判定 = rawScore を閾値で分類（下記参照）
```

### 最終判定閾値

| rawScore | 記号 | スコア値 |
|---|---|---|
| **0.8 以下** | ◎ 釣り日和 | 3 |
| **0.8 より大〜1.2 以下** | ○ 出船可能 | 2 |
| **1.2 より大〜1.6 以下** | △ 未定・要確認 | 1 |
| **1.6 より大** | × 出船不可 | 0 |

---

### ファクター① 波高ベース値

室戸沖での遊漁船（約40フィート）の運航可否を波高メートル値で評価するベース値。  
変換せず、実際の波高値（m）をそのまま計算のベースとして使用する。

---

### ファクター② 風向補正（室戸岬地形特性）

室戸岬沖は南〜南東方向から風が吹くと沖合の波が成長しやすい地形特性がある。  
北風（陸からの風）は比較的安全。悪条件では rawScore が増加（判定が悪化）する。

| 条件 | 補正値 | タグ表示 |
|---|---|---|
| 風向 **135°〜225°**（南〜南東〜南西）かつ風速 **> 6 m/s** | **+0.5** | 南風↑（赤） |
| 風向 **315°〜360° または 0°〜45°**（北〜北東〜北西） | ±0 | 北風◎（緑） |
| 上記以外（東系・西系） | **+0.25** | 東西風（橙） |

> 風向の 0° = 北（真北）、90° = 東、180° = 南、270° = 西

---

### ファクター③ 逆波補正（風向 vs 波向きの干渉）

風の向きと波の進行方向が逆・直交する場合、三角波・急峻波が発生し、同じ波高でも体感リスクが増大する。

| 風向と波向きの角度差 | 補正値 | タグ表示 |
|---|---|---|
| **120° 以上**（風と波がほぼ逆方向） | **+0.2** | 逆波（赤） |
| **90°〜120°**（交差方向） | **+0.1** | 交差波（橙） |
| **90° 未満**（同方向系） | ±0 | — |

角度差の計算:
```
diff = min(|windDir - waveDir| mod 360, 360 - |windDir - waveDir| mod 360)
```

> 波向きデータ（`wave_direction`）が取得できない場合、このファクターは省略。

---

### ファクター④ グレーゾーン複合補正

グレーゾーン波高帯（0.8m超）で、南風補正と逆波補正が**同時に発生**した場合の追加悪化補正。  
「波高は中程度だが悪条件が重なっている」状況を正確に評価する。

| 適用条件 | 補正値 | タグ表示 |
|---|---|---|
| wave > 0.8m **かつ** ②南風補正発動 **かつ** ③逆波または交差波補正発動 | **+0.2** | 複合悪化（紫） |

---

### ファクター⑤ 風速トレンド補正

前スロット（6時間前）との風速変化を評価。風が強まっている場合は危険側補正、弱まっている場合は安全側補正。

| 条件 | 補正値 | タグ表示 |
|---|---|---|
| 現在の風速 > 前スロット + **0.5 m/s**（風速増加傾向） | **+0.2** | 風↑（橙） |
| 現在の風速 < 前スロット − **0.5 m/s**（風速減少傾向） | **−0.2** | 風↓（緑） |
| 変化量 ±0.5 m/s 未満 | ±0 | — |

> 「前スロット」は同日の直前抽出時刻のデータ。0時スロットは前日18時のデータを参照。

---

### ファクター⑥ 風波補正

風波（`wind_wave_height`）は短周期の急峻な波。小さければ安全側、大きければ危険側に補正。

| 条件 | 補正値 | タグ表示 |
|---|---|---|
| 風波 **≤ 0.3 m**（風波小） | **−0.3** | 風波小（緑） |
| 風波 **≥ 0.7 m**（風波大） | **+0.3** | 風波大（赤） |
| 0.3m 超〜0.7m 未満 | ±0 | — |

> 風波データ（`wind_wave_height`）が取得できない場合、このファクターは省略。

## 3. 最終スコアと出船判定

| スコア | 記号 | 意味 | 予約システム送信ステータス |
|---|---|---|---|
| **3** | ◎ 釣り日和 | 海況良好・積極的に出船可 | ✅ 出船可 |
| **2** | ○ 出船可能 | 出船可能・通常運航 | ✅ 出船可 |
| **1** | △ 未定・要確認 | 船長判断要・予約者に要連絡 | ⚠ 要確認 |
| **0** | × 出船不可 | 安全上の理由で出船不可 | ❌ 中止 |

---

## 4. 日次総合判定の計算方法

各日の総合判定 = **(6:00 rawScore + 12:00 rawScore) ÷ 2** を閾値分類

- 6:00 = 出港時の海況
- 12:00 = 沖での最も活動的な時間帯の海況

両スロットの rawScore 平均値を代表値として使用し、②の最終判定閾値で分類する。

## 6. 注意事項・制約

1. **`wave_height` は合成波高**（風波＋うねりの総和）。うねりが卓越する海況では実際より波高を過小評価する場合がある。
2. **予測精度は時間とともに低下**。3日目以降の判定は参考値として扱う。
3. **船長判断が最優先**。本システムの出力はあくまで参考情報であり、最終判断は船長が行う。
4. **突発的な気象変化**（急発達低気圧など）はモデル予測に遅れが生じる場合がある。
5. **波向きデータ**が Open-Meteo から取得できない場合、ファクター③④は省略される。
6. **風波データ**が Open-Meteo から取得できない場合、ファクター⑥は省略される。
